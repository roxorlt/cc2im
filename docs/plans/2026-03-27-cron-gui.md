# Cron Scheduler GUI 实现计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在 Dashboard 中添加定时任务管理页面，支持查看/创建/删除定时任务。

**Architecture:** server.ts 新增 REST API 直接调用 cron DB 函数；前端新增 ScheduledTasksPage 组件，遵循 ChannelsPage 的卡片+对话框模式。

**Tech Stack:** React（inline styles + CSS variables）、fetch API

---

## Task 1: Server API endpoints

**Files:**
- Modify: `src/plugins/web-monitor/server.ts`

新增 4 个 REST endpoints，放在现有 `/api/` 路由块中：

- `GET /api/cron-jobs` — 返回所有 cron jobs（含最近 5 条 runs）
- `POST /api/cron-jobs` — 创建任务（body: name, agentId, scheduleType, scheduleValue, timezone, message）
- `DELETE /api/cron-jobs/{id}` — 删除任务
- `PATCH /api/cron-jobs/{id}` — 更新任务（enable/disable、修改字段）

直接 import cron DB 函数（同进程共享 SQLite）：
```typescript
import { openCronDb, createJob, deleteJob, updateJob, listJobs, getRecentRuns } from '../cron-scheduler/db.js'
import { CronScheduler } from '../cron-scheduler/scheduler.js'
```

POST 时需要用 CronScheduler.calcNextRun 计算 nextRun。

**Commit:** `feat(cron-gui): add REST API for cron job management`

## Task 4: ScheduledTasksPage 组件

**Files:**
- Create: `src/plugins/web-monitor/frontend-v2/components/ScheduledTasksPage.tsx`

遵循 ChannelsPage 的模式：

**TaskCard 组件：**
- 显示：名称、调度类型+表达式、目标 agent、enabled 状态灯、下次执行时间、最近执行状态

**AddTaskDialog 组件（Modal）：**

- 创建按钮调 POST /api/cron-jobs

**样式：** 与 ChannelsPage 一致的 CSS variables + inline styles。
